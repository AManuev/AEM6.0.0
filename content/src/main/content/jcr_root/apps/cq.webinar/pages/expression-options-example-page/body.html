Expression options example page.


1. String formatting.

${'Page with title "{0}".' @ format=[currentPage.title]}

2. I18n.

${'Page' @ i18n}

AND

${'Page' @ i18n, locale='fr'}

AND

${'Page' @ i18n, source='user'}


3. Array join.

${['one', 'two', 'three'] @ join='; '}

4. Display context.

Here we have three variables in three different contexts: properties.link (uri context), properties.title (attribute context) and properties.text (text context). Sightly will escape each of these differently in accordance with the security requirements of theirs respective contexts. No explicit context settings is required in normal cases such as this one:

${properties.text}
Explicit context: HTML:
${properties.richText @ context='html'}

Explicit context: Style:
Styling string
Explicit context: Script:
Script string
Explicit context: Off:
${properties.richText @ context='unsafe'}
